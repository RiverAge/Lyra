# Lyra 前端设计系统

> 适用范围:所有 Vue 组件的视觉决策。token 是唯一真源,不自创色值/间距/动画。
> 提示词引用:写组件任务时在前置阅读里列本文档,不重述内容(见 §7)。
> 风格参照:Linear / Notion 极简风(大量留白、低饱和中性色 + 单一强调色、细边框、微阴影)。
> 机器落地:token 值将来在 `frontend/src/styles/tokens.css`(Tailwind v4 `@theme` 块)实现,前端骨架任务时创建。本文档是 token 的规格定义,与该文件保持同步。

---

## 1. 设计原则

1. **克制**:留白优先于装饰。优先级序:留白 > 间距 > 边框 > 阴影 > 色块。能用留白解决就不加边框,能用边框解决就不加阴影。
2. **单一强调**:全局只用一个 accent 色(主按钮/选中态/链接)。其余 UI 走中性色阶(bg-surface/text-primary/border-default),不引入第二个强调色。
3. **语义 token**:组件用语义名(`bg-surface`/`text-primary`/`border-subtle`),不写具体十六进制。主题切换时只改变量定义,不改组件代码。
4. **动画服务于反馈**:过渡用于状态变化反馈(出现/消失/hover/选中),不做纯装饰动画。loading 用骨架屏,不用循环 animation。

## 2. 设计 Token 定义

### 2.1 色彩(语义层)

| Token | 亮色 | 暗色 | 用途 |
|---|---|---|---|
| `--color-bg-base` | `#ffffff` | `#0a0a0a` | 页面最底层背景 |
| `--color-bg-surface` | `#ffffff` | `#141414` | 卡片/面板背景 |
| `--color-bg-subtle` | `#f7f7f8` | `#1c1c1e` | 悬浮/选中行底色 |
| `--color-bg-hover` | `#f0f0f2` | `#262629` | hover 态底色 |
| `--color-border-default` | `#e5e5e7` | `#2a2a2d` | 默认边框/分割线 |
| `--color-border-subtle` | `#f0f0f2` | `#1f1f22` | 弱分割线 |
| `--color-text-primary` | `#1d1d1f` | `#f5f5f7` | 主文字 |
| `--color-text-secondary` | `#6e6e73` | `#a1a1a6` | 次文字/标签 |
| `--color-text-tertiary` | `#a1a1a6` | `#6e6e73` | 占位/禁用 |
| `--color-accent` | `#5b8def` | `#5b8def` | 强调色(主按钮/选中/链接) |
| `--color-accent-hover` | `#4a7fd9` | `#6b9bf2` | accent hover |
| `--color-accent-subtle` | `#eaf1ff` | `#1a2536` | accent 淡底(选中行) |
| `--color-success` | `#34c759` | `#30d158` | 成功 |
| `--color-warning` | `#ff9500` | `#ff9f0a` | 警告 |
| `--color-danger` | `#ff3b30` | `#ff453a` | 错误/删除 |

> accent 色 `#5b8def` 亮暗一致——强调色是品牌色,不随主题变。hover/subtle 态分别调深/调浅。

### 2.2 字号阶梯

固定 px,不用流式(rem/vw)。4 步够用,不引入更多档。

| Token | 值 | 用途 |
|---|---|---|
| `--font-size-sm` | 13px | 辅助文字/标签/元信息 |
| `--font-size-base` | 14px | 正文(默认) |
| `--font-size-lg` | 16px | 小标题/强调文字 |
| `--font-size-xl` | 20px | 区块标题/页面标题 |

行高:`--line-height-sm: 1.5` / `--line-height-base: 1.6` / `--line-height-lg: 1.5`

字重:`--font-weight-normal: 400` / `--font-weight-medium: 500` / `--font-weight-semibold: 600`

> 不用 `bold`(700)。强调用 `semibold`(600),不用加粗。

### 2.3 间距(8 网格)

所有间距是 4 的倍数。Tailwind 原生 `p-*/gap-*/m-*` 已覆盖,不自创间距值。

| Token | 值 | Tailwind 类 |
|---|---|---|
| `--space-1` | 4px | `p-1` / `gap-1` / `m-1` |
| `--space-2` | 8px | `p-2` / `gap-2` |
| `--space-3` | 12px | `p-3` / `gap-3` |
| `--space-4` | 16px | `p-4` / `gap-4` |
| `--space-6` | 24px | `p-6` / `gap-6` |
| `--space-8` | 32px | `p-8` / `gap-8` |

> 不用 5px/7px/10px 等非 4 倍数值。Tailwind 原生不支持的不用。

### 2.4 圆角

| Token | 值 | 用途 |
|---|---|---|
| `--radius-sm` | 6px | 按钮/标签/小卡片/input |
| `--radius-md` | 8px | 卡片/面板/模态 |
| `--radius-full` | 9999px | 头像/胶囊/圆角标签 |

> 不用 4px(太方)/12px(太圆)。2 档 + full 够用。

### 2.5 阴影

极克制,只 2 级。Linear/Notion 风的核心是"几乎无阴影",靠边框和留白分层。

| Token | 亮色 | 暗色 | 用途 |
|---|---|---|---|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.04)` | `0 1px 2px rgba(0,0,0,0.3)` | 默认/卡片边 |
| `--shadow-md` | `0 4px 12px rgba(0,0,0,0.08)` | `0 4px 12px rgba(0,0,0,0.4)` | 悬浮/模态/dropdown |

> 暗色模式阴影 rgba alpha 加深到 0.3/0.4(暗底上浅阴影看不见)。

## 3. 主题切换(亮/暗)

- **机制**:CSS 变量 + `data-theme` 属性根切换
- 根元素 `<html>` 挂 `data-theme="light"` 或 `data-theme="dark"`
- `:root[data-theme="light"]` 定义亮色 `--color-*` 变量集
- `:root[data-theme="dark"]` 覆盖为暗色 `--color-*` 变量集
- **不用 `prefers-color-scheme` 自动跟随**(v1 手动切换,用户自选;避免"白天暗色"的困扰)
- 切换逻辑放 `composables/useTheme.ts`(后续建),本文档只定 token 映射
- 默认主题:`light`(首次进入)

## 4. 标准动画集

### 4.1 6 个标准动画

| 名称 | 时长 | 曲线 | 用途 | 实现 |
|---|---|---|---|---|
| `fade-in` | 150ms | `ease-out` | 元素出现(toast/dropdown) | opacity 0→1 |
| `fade-out` | 100ms | `ease-in` | 元素消失 | opacity 1→0 |
| `slide-up` | 200ms | `cubic-bezier(0.16,1,0.3,1)` | 模态/抽屉入场 | translateY(8px)→0 + opacity |
| `scale-in` | 150ms | `ease-out` | popover/菜单 | scale(0.96)→1 + opacity |
| `slide-down` | 200ms | `cubic-bezier(0.16,1,0.3,1)` | 下拉菜单展开 | translateY(-8px)→0 + opacity |
| `progress` | 300ms | `ease-out` | 切换/加载进度 | width/opacity |

> `cubic-bezier(0.16,1,0.3,1)` 是"ease-out-back"变体,出场有轻微减速过冲感,适合入场动画。

### 4.2 性能约束(强制)

- **只动 `transform` 和 `opacity`**,不动 `width/height/top/left/margin/padding`(触发 layout 重排)
- 唯一例外:`progress` 动画可用 width(进度条场景不可避免)
- 动画时长区间:消失态 100ms / 出现态 150-200ms / 大元素(模态/抽屉)≤300ms
- 超过 300ms 的动画必须在代码注释标注理由
- 不用 `animation` 做循环动画(loading 骨架屏除外),循环加载用骨架屏
- `prefers-reduced-motion: reduce` 时所有动画降为 0ms(无障碍,`tokens.css` 全局处理)

## 5. 组件视觉规范

### 5.1 布局

- 默认间距:卡片内 padding `p-4`(16px),卡片间 `gap-4`(16px)
- 页面级:max-width 限制内容宽度(列表页 1200px / 详情页 900px),两侧 `mx-auto` 居中
- 页面外边距:移动端 `p-4`,桌面端 `p-6` 起

### 5.2 边框与分割

- 默认边框:1px solid `border-default`
- 弱分割(列表行间/区块内):`border-subtle`
- 不用 2px+ 粗边框(除选中态左侧色条)

### 5.3 交互态

| 态 | 表现 | 禁止 |
|---|---|---|
| hover | 背景变 `bg-hover` | 不变色/不放大/不加阴影/不位移 |
| 选中 | 背景变 `accent-subtle` + 左侧 2px `accent` 色条 | 不变色/不加粗 |
| 禁用 | opacity 0.5 + `cursor-not-allowed` | 不变色(保留原色,只降透明) |
| focus | 2px `accent` 色 outline,offset 2px | 不用 box-shadow 模拟(无障碍) |
| active(按下) | 背景再深一档(比 hover 更暗) | 不位移/不缩放 |

### 5.4 按钮

| 尺寸 | 高度 | 横向 padding | 字号 |
|---|---|---|---|
| sm | 32px | `px-3` | `text-sm`(13px) |
| md | 36px | `px-4` | `text-base`(14px) |
| lg | 40px | `px-4` | `text-base`(14px) |

- 主按钮:`bg-accent` + `text-white` + `hover:bg-accent-hover`
- 次按钮:`bg-surface` + `border-default` + `text-primary` + `hover:bg-hover`
- 危险按钮:`bg-danger` + `text-white`
- 不用渐变按钮/阴影按钮/图标+文字混排的"大按钮"

## 6. Tailwind v4 映射

token 的机器落地方式(前端骨架任务时实现):

- v4 用 CSS `@theme` 块定义 token,自动生成对应工具类
- token 落地文件:`frontend/src/styles/tokens.css`(将来创建)
- 示例映射:`--color-bg-surface` → `bg-surface` / `--color-text-primary` → `text-primary` / `--color-border-default` → `border-default`
- 语义色通过 `@theme` 的 `--color-*` 命名注册,组件直接写 `bg-surface`/`text-primary`/`border-default`
- 亮/暗主题:`tokens.css` 定义 `:root[data-theme=light]` 和 `:root[data-theme=dark]` 两套 `--color-*` 值,`@theme` 块引用变量
- 字体:`--font-sans` / `--font-mono` 在 `@theme` 块注册(从 `tailwind.config.js` 迁移)
- 动画:`@theme` 注册 `--animate-*` token,`@keyframes` 在 `tokens.css` 底部定义
- 接入链路:已定为 PostCSS(`@tailwindcss/postcss`),不走 `@tailwindcss/vite` 插件
