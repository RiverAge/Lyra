# Lyra 前端设计系统

> 适用范围:所有 Vue 组件的视觉决策。token 是唯一真源,不自创色值/间距/动画。
> 提示词引用:写组件任务时在前置阅读里列本文档,不重述内容(见 §7)。
> 风格参照:Linear / Vercel 浅色克制风(大量留白、zinc 中性灰阶 + 墨黑单一强调色、实色发丝线、几乎无阴影)。
> 机器落地:token 值在 `frontend/src/styles/tokens.css`(Tailwind v4 `@theme` 块)实现。本文档是 token 的规格定义,与该文件保持同步——**代码是真源,本文档随代码反向同步**。

---

## 0. 设计基调

**浅色克制,墨黑即强调色。**

- 暖白底(非纯白)+ zinc 灰阶文字 + **墨黑作为唯一强调色**(克制到极致,不靠彩色博眼球)
- 实色发丝线分层,几乎无阴影(Linear/Vercel 核心是"靠边框和留白分层,不靠阴影")
- 圆角克制(button/input sm,card lg)
- **禁渐变、禁发光 glow**
- **单一浅色主题,无暗色分支**(见 §3)

> 设计意图:B 方案(Linear/Vercel 流派)。强调不靠彩色,靠墨黑与留白的对比建立层级。

---

## 1. 设计原则

1. **克制**:留白优先于装饰。优先级序:留白 > 间距 > 边框 > 阴影 > 色块。能用留白解决就不加边框,能用边框解决就不加阴影。
2. **单一强调**:全局只用一个 accent 色(墨黑)。主按钮/选中态/链接/播放控件用 `accent`;其余 UI 走中性灰阶(`bg-surface`/`text-primary`/`border-line`),不引入第二个强调色。
3. **语义 token**:组件用语义名(`bg-surface`/`text-primary`/`border-line`),不写具体十六进制。
4. **动画服务于反馈**:过渡用于状态变化反馈(出现/消失/hover/选中),不做纯装饰动画。loading 用骨架屏,不用循环 animation。
5. **禁渐变禁发光**:不使用 `linear-gradient` 做按钮/背景;focus 态用边框换色 + 极轻 ring,不用发光 box-shadow。

## 2. 设计 Token 定义

### 2.1 色彩(语义层)

单一浅色主题,无亮/暗双值。所有值为浅色基调。

| Token | 值 | 用途 |
|---|---|---|
| `--color-base` | `#fcfcfd` | 页面最底层背景(暖白,非纯白) |
| `--color-surface` | `#ffffff` | 卡片/面板背景(纯白) |
| `--color-subtle` | `#f7f7f9` | 表头/分页栏(比 base 更沉) |
| `--color-hover` | `#f3f3f5` | hover 态底色 |
| `--color-line` | `#e8e8ec` | 默认边框/分割线(实色发丝线) |
| `--color-line-subtle` | `#f0f0f3` | 弱分割线(列表行间/区块内) |
| `--color-line-strong` | `#d4d4dc` | hover/focus 加深边框 |
| `--color-primary` | `#18181b` | 主文字(zinc-900) |
| `--color-secondary` | `#71717a` | 次文字/标签(zinc-500) |
| `--color-tertiary` | `#a1a1aa` | 占位/禁用(zinc-400) |
| `--color-accent` | `#18181b` | 强调色(墨黑,主按钮/选中/链接/播放控件) |
| `--color-accent-hover` | `#000000` | accent hover(纯黑) |
| `--color-accent-pressed` | `#27272a` | accent 按下 |
| `--color-accent-subtle` | `#f3f3f5` | accent 淡底(选中行/徽章) |
| `--color-on-accent` | `#ffffff` | accent 背景上的文字色 |
| `--color-success` | `#16a34a` | 成功 |
| `--color-success-subtle` | `rgba(22,163,74,0.10)` | 成功淡底 |
| `--color-warning` | `#d97706` | 警告 |
| `--color-warning-subtle` | `rgba(217,119,6,0.10)` | 警告淡底 |
| `--color-danger` | `#dc2626` | 错误/删除 |
| `--color-danger-subtle` | `rgba(220,38,38,0.10)` | 错误淡底 |

> **accent 即墨黑**(`#18181b`,与 text-primary 同值)。这是 B 方案的核心取舍:强调色不靠彩色,靠墨黑与留白的对比。`on-accent` 为 accent 背景上的白字。`accent-subtle` 复用 hover 底色(墨黑没有"淡彩色"概念,淡底用中性灰)。

### 2.2 字号阶梯

固定 px,不用流式(rem/vw)。5 步。

| Token | 值 | 用途 |
|---|---|---|
| `--font-size-sm` | 13px | 辅助文字/标签/元信息 |
| `--font-size-base` | 14px | 正文(默认) |
| `--font-size-lg` | 16px | 小标题/强调文字 |
| `--font-size-xl` | 20px | 区块标题 |
| `--font-size-2xl` | 24px | 大标题 |

行高:`--line-height-sm: 1.5` / `--line-height-base: 1.6` / `--line-height-lg: 1.5`

字重:`--font-weight-normal: 400` / `--font-weight-medium: 500` / `--font-weight-semibold: 600`

> 不用 `bold`(700)。强调用 `semibold`(600)。

### 2.3 间距(8 网格)

所有间距是 4 的倍数。Tailwind 原生 `p-*/gap-*/m-*` 已覆盖,不自创间距值。

| Tailwind 类 | 值 |
|---|---|
| `p-1` / `gap-1` / `m-1` | 4px |
| `p-2` / `gap-2` | 8px |
| `p-3` / `gap-3` | 12px |
| `p-4` / `gap-4` | 16px |
| `p-6` / `gap-6` | 24px |
| `p-8` / `gap-8` | 32px |

> 不用 5px/7px/10px 等非 4 倍数值。Tailwind 原生不支持的不用。

### 2.4 圆角

4 档 + full。

| Token | 值 | 用途 |
|---|---|---|
| `--radius-sm` | 6px | 按钮/标签/小卡片/input |
| `--radius-md` | 8px | 中等卡片/内嵌表格 |
| `--radius-lg` | 12px | 卡片/面板/模态 |
| `--radius-full` | 9999px | 头像/胶囊/圆角标签/pill |

### 2.5 阴影

极克制,5 级。B 方案核心是"几乎无阴影",靠发丝线和留白分层。

| Token | 值 | 用途 |
|---|---|---|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.03)` | 默认/细元素 |
| `--shadow-md` | `0 2px 8px rgba(0,0,0,0.04)` | 中等悬浮 |
| `--shadow-card` | `0 1px 2px rgba(0,0,0,0.03)` | 卡片默认(= sm) |
| `--shadow-lg` | `0 4px 16px rgba(0,0,0,0.06)` | 悬浮/dock/modal |
| `--shadow-glow` | `0 0 0 1px var(--border-line-strong), 0 0 0 3px rgba(24,24,27,0.08)` | focus ring(边框换色 + 极轻 ring,**非发光**) |

> `shadow-glow` 名字沿用历史,语义已变:不再是彩色发光,而是 focus 时的边框加深 + 极轻墨黑 ring(无障碍 focus 指示)。`.input-ring:focus` 用此 token。

## 3. 主题(单一浅色,无暗色)

- **单一浅色主题**,无亮/暗切换
- token 直接定义在 `:root`(无 `data-theme` 分支)
- **不挂 `data-theme` 属性**,html 元素不带主题属性
- 历史的亮/暗双主题 + `data-theme` 切换已废弃(见 `stores/app.ts`:主题逻辑已删,store 改纯占位)

> 设计取舍:B 方案浅色克制与暗色"克制到极致"哲学冲突,且单用户自用场景无需暗色。如未来要恢复暗色,在 `:root[data-theme="dark"]` 补一套值并恢复 app store 的切换逻辑。

## 4. 标准动画集

### 4.1 动画 token

| 名称 | 时长 | 曲线 | 用途 | keyframes |
|---|---|---|---|---|
| `--animate-slide-up` | 200ms | `cubic-bezier(0.16,1,0.3,1)` | 模态/抽屉/卡片入场 | translateY(8px)→0 + opacity |
| `--animate-fade` | 200ms | `ease-out` | 元素出现(toast/dropdown/入场) | opacity 0→1 |
| `--animate-progress` | 300ms | `ease-out` | 切换/加载进度 | width/opacity(进度条唯一例外) |

动画时长 token:`--animate-duration-slide: 200ms` / `--animate-duration-progress: 300ms` / `--animate-duration-hover: 150ms`

页面切换过渡:`.page-enter/leave-active` 0.18s,opacity + translateY(±10px),曲线 `cubic-bezier(0.4,0,0.2,1)`。

`spin` keyframes(loading 图标旋转)保留,供 Icon `Loader2` 用。

> `cubic-bezier(0.16,1,0.3,1)` 是"ease-out-back"变体,出场轻微减速过冲,适合入场动画。

### 4.2 性能约束(强制)

- **只动 `transform` 和 `opacity`**,不动 `width/height/top/left/margin/padding`(触发 layout 重排)
- 唯一例外:`progress` 动画可用 width(进度条场景不可避免)
- 动画时长区间:消失态 100ms / 出现态 150-200ms / 大元素(模态/抽屉)≤300ms
- 超过 300ms 的动画必须在代码注释标注理由
- 不用 `animation` 做循环动画(loading 骨架屏 + `spin` 图标除外)
- `prefers-reduced-motion: reduce` 时所有动画降为 0.01ms(无障碍,`tokens.css` 全局处理)

## 5. 组件视觉规范

### 5.1 布局

- 默认间距:卡片内 padding `p-4`~`p-5`(16-20px),卡片间 `gap-4`(16px)
- 页面级:max-width 限制内容宽度(列表页 `max-w-6xl`/1100px / 详情页 `max-w-4xl`~`max-w-3xl`),两侧 `mx-auto` 居中
- 页面外边距:桌面端 `px-6 py-8` 起

### 5.2 边框与分割

- 默认边框:1px solid `border-line`
- 弱分割(列表行间/区块内):`border-line-subtle`
- hover/focus 加深:`border-line-strong`
- 不用 2px+ 粗边框(除选中态左侧色条)

### 5.3 交互态

| 态 | 表现 | 禁止 |
|---|---|---|
| hover | 背景变 `bg-hover` / 边框变 `border-line-strong` | 不变色/不放大/不加阴影/不位移 |
| 选中 | 背景变 `accent-subtle` + 左侧 2px `accent` 色条 | 不变色/不加粗 |
| 禁用 | opacity 0.4~0.5 + `cursor-not-allowed` | 不变色(保留原色,只降透明) |
| focus | 边框换 `accent` + `shadow-glow`(极轻 ring) | 不用纯彩色发光 |
| active(按下) | 背景再深一档 | 不位移/不缩放 |

> **card-hover 不浮起**:B 方案去掉 translateY 浮起,hover 只变边框 + 底色(克制)。

### 5.4 按钮

| 尺寸 | 横向 padding | 字号 |
|---|---|---|
| sm | `px-2.5` | `text-xs`(13px) |
| md | `px-3.5` | `text-sm`(14px) |

- 主按钮(primary):`bg-accent`(墨黑)+ `text-on-accent`(白)+ hover `bg-accent-hover`
- 次按钮(secondary):`bg-surface` + `border-line` + `text-secondary` + hover `bg-hover`/`text-primary`
- 幽灵按钮(ghost):透明 + `text-secondary` + hover `bg-hover`/`text-primary`
- 危险按钮(danger):叠加在 primary/secondary 上,改用 `bg-danger`/`bg-danger-subtle` + `text-danger`
- **不用渐变按钮**(`.bg-accent-gradient` 工具类保留但指向纯墨色,仅为兼容历史类名)

### 5.5 卡片(工具类)

| 类 | 用途 |
|---|---|
| `.card` | 基础卡片:白底 + `border-line` + `shadow-card` + `radius-lg` |
| `.card-hover` | 可交互卡片:hover 变 `bg-hover` + `border-line-strong`(不浮起) |
| `.card-elevated` | 最高层(dock/modal/浮层):白底 + `border-line-strong` + `shadow-lg`(靠深边框分层) |

### 5.6 输入框

- `.input-ring`:focus 时边框换 `accent` + `0 0 0 3px rgba(24,24,27,0.08)` 极轻 ring(去发光)
- 圆角 `radius-sm`,边框 `border-line-subtle`,底 `bg-surface`

### 5.7 表格(B 范式)

LibraryView/TrackDetail 的字段表/diff 表统一用 **grid 列**范式(非 `<table>`):

- `.track-row` / `.table-head`:grid-template-columns 定义列宽,发丝线 `border-line-subtle` 分隔
- 表头:`bg-subtle` + uppercase 11px + `text-tertiary`
- 行 hover:`bg-hover`
- 数字列:`font-variant-numeric: tabular-nums` + `font-mono`

## 6. Tailwind v4 映射

token 的机器落地方式:

- v4 用 CSS `@theme` 块定义 token,自动生成对应工具类
- token 落地文件:`frontend/src/styles/tokens.css`
- 示例映射:`--color-surface` → `bg-surface` / `--color-primary` → `text-primary` / `--color-line` → `border-line`
- 语义色通过 `@theme` 的 `--color-*` 命名注册,组件直接写 `bg-surface`/`text-primary`/`border-line`
- **单一主题**:`tokens.css` 直接在 `:root` 定义 `--color-*` 值(无 data-theme 分支),`@theme` 块引用变量
- 字体:`--font-sans`(Inter + Noto Sans SC)/ `--font-mono`(JetBrains Mono)在 `@theme` 块注册
- 动画:`@theme` 注册 `--animate-*` token,`@keyframes` 在 `tokens.css` 底部定义
- 接入链路:PostCSS(`@tailwindcss/postcss`),不走 `@tailwindcss/vite` 插件
